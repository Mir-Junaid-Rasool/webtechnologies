üìä The Display Property

The display property controls how an element is displayed and how it behaves in the layout. It's one of the most important CSS properties for controlling page layout.

Value Behavior Default For
block Takes full width, starts on new line div, p, h1-h6, section
inline Takes only needed width, stays in line span, a, strong, em
inline-block Inline but respects width/height img, button
none Default - no float
Float Example (Text wrapping around image):
img { float: left; margin: 0 20px 20px 0; }
Image

Lorem ipsum dolor sit amet, consectetur adipiscing elit. This text wraps around the floated element. The float property removes the element from normal flow and allows other content to wrap around it. This is particularly useful for magazine-style layouts where text flows around images. However, for modern layouts, Flexbox and Grid are preferred over floats.

Clear Property:

The clear property prevents elements from wrapping around floats.

.clear-both { clear: both; /* Clears both left and right floats */ }
Value Description
left No floating elements on left
right No floating elements on right
both No floating elements on either side
none Default - allows floating elements
üí° Modern Approach:
For layouts, use Flexbox or Grid instead of floats. Floats are now mainly used for their original purpose: wrapping text around images.

‚ö†Ô∏è Common Mistakes & Tips

‚ùå Common Mistakes:

1. Using width/height on inline elements:
They won't work! Use display: inline-block instead.

2. Forgetting position: relative on parent:
Absolute positioned children need a positioned parent.

3. Z-index not working:
Remember: z-index only works on positioned elements (not static).

4. Float layout issues:
Don't forget to clear floats or use modern Flexbox/Grid instead.

5. Fixed positioning covering content:
Add padding-top to body equal to fixed header height.
‚úÖ Best Practices:

‚Ä¢ Use Flexbox/Grid for layouts instead of floats
‚Ä¢ Use position: relative as parent for absolute children
‚Ä¢ Keep z-index values organized (100s, 1000s for different layers)
‚Ä¢ Use display: inline-block for inline elements that need dimensions
‚Ä¢ Test fixed/sticky elements on different screen sizes
‚Ä¢ Use semantic HTML with proper display values
‚Ä¢ Remember: absolute/fixed elements are removed from flow

Block elements:

  • Take up the full width available
  • Start on a new line
  • Stack vertically
  • Respect width, height, margin, padding
.block-element { display: block; width: 200px; height: 50px; background: #3498db; margin: 10px 0; }

Inline elements:

  • Take only the width they need
  • Stay on the same line with other inline elements
  • Do NOT respect width/height properties
  • Only respect horizontal margin/padding
.inline-element { display: inline; background: #3498db; padding: 5px 10px; }

  • Perfect for horizontal layouts

üìç The Position Property

The position property specifies the positioning method for an element. It works with the properties: top, right, bottom, left.

Value Behavior Use Case
static Default - normal document flow Default positioning
relative Positioned relative to itself Small adjustments, parent for absolute
absolute Positioned relative to positioned parent Overlays, tooltips, dropdowns
fixed Positioned relative to viewport Fixed headers, back-to-top buttons
sticky Hybrid of relative and fixed Sticky navigation bars

1Ô∏è‚É£ position: static (Default)

Elements are positioned according to normal document flow. top, right, bottom, left have NO effect.

.static-box { position: static; /* Default value */ }

2Ô∏è‚É£ position: relative

Positioned relative to its normal position:

  • Element stays in normal flow
  • Can be offset using top, right, bottom, left
  • Original space is preserved
  • Often used as parent for absolute positioned children
.relative-box { position: relative; top: 20px; /* Moved down 20px */ left: 30px; /* Moved right 30px */ }

3Ô∏è‚É£ position: absolute

Positioned relative to nearest positioned ancestor:

  • Removed from normal document flow
  • Other elements act like it doesn't exist
  • Positioned relative to nearest parent with position (not static)
  • If no positioned parent, uses the document body
.parent { position: relative; /* Creates positioning context */ } .absolute-child { position: absolute; top: 10px; right: 10px; }

4Ô∏è‚É£ position: fixed

Positioned relative to the viewport (browser window):

  • Stays in same position even when scrolling
  • Removed from normal flow
  • Perfect for navigation bars, floating buttons
.fixed-header { position: fixed; top: 0; left: 0; width: 100%; z-index: 1000; }
üí° Common Uses:
‚Ä¢ Fixed navigation bars at top
‚Ä¢ Floating "Back to Top" buttons
‚Ä¢ Chat widgets
‚Ä¢ Cookie consent banners

5Ô∏è‚É£ position: sticky

Hybrid between relative and fixed:

  • Acts like relative until scroll threshold
  • Then becomes fixed at specified position
  • Great for section headers
.sticky-header { position: sticky; top: 0; /* Sticks when reaching top */ }

üìö z-index Property

The z-index property controls the stacking order of positioned elements. Higher values appear on top of lower values.

Important points:

  • Only works on positioned elements (not static)
  • Higher z-index = closer to viewer
  • Can be negative
  • Default is auto (0)
.bottom { position: relative; z-index: 1; } .middle { position: relative; z-index: 2; } .top { position: relative; z-index: 3; }
